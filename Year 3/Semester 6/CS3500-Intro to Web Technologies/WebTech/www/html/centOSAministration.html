Note: The "$" refers to the command's prompt. Don't type in the $ when you copy these commands. Also, you must be logged in as root to do most of these commands.

Changing your password(for root or for a user)

#First thing to do is Change your password. Do this with
$ passwd

Adding a user

#Type
$ useradd
# is the name of the user you'd like to create
#Now you need to activate it by setting a passwd. Do this with
$ passwd
#You can also use this command to change other users' passwords from root

Creating a group and adding users to it

#Use this to create a group with the name
$ groupadd
#Now add a user to that group
$ usermod -a -G

Adding a user to the sudoers list

#Find and open the sudoers file in
/etc/sudoers
#At the very bottom, add this line:
ALL=(ALL) ALL
#This will give the permission to use the sudo command with their personal password.

ssh

ssh requires two things: a public key and a private key. These are made on seperate machines. The public key is made on the server machine, while the private key is made on the connecting machine. The public key must be on the connecting key, and the private key must be on the server machines(this requires simply copying and pasting the files into the /.ssh/authorized_keys file.
When using pretty much any modern machine, this is done automatically and usually prompts the user for a logon password. To do this without passwords, both machines need each others keys

Disable Root Logins

#Open the sshd config file located here:
/etc/ssh/sshd_config
#Make sure you have the following entry
PermitRootLogin no
#Now restart the sshd service with
$ service sshd restart
